## üí° Tips for Presenters

### General Guidelines
- **Pace:** Speak clearly and not too fast
- **Pauses:** Add natural pauses for comprehension
- **Energy:** Maintain enthusiasm throughout
- **Clarity:** Explain technical terms when first used

### Screen Recording
- **Resolution:** 1920x1080 minimum
- **Font Size:** Large enough for mobile viewing
- **Highlighting:** Use cursor or annotations to highlight important areas
- **Cuts:** Edit out mistakes and long waits

### Code Examples
- **Type vs Paste:** Consider typing for teaching, pasting for efficiency
- **Explain:** Describe what code does before running
- **Results:** Always show the output/result
- **Errors:** Show how to debug common errors

### Engagement
- **Questions:** Ask rhetorical questions to engage viewers
- **Summaries:** Recap at the end of each section
- **Next Steps:** Always point to what's next
- **CTAs:** Encourage likes, subscribes, and discussions
